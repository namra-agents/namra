## Key Architectural Decisions

### 1. **Why Rust?**
- **Performance**: 10-100Ã— faster than Python
- **Memory Safety**: No null pointers, no data races
- **Concurrency**: Tokio async runtime
- **Single Binary**: Easy deployment

### 2. **Why Config-Driven?**
- **Accessibility**: Non-programmers can create agents
- **GitOps**: Version control YAML files
- **Hot Reload**: No recompilation needed
- **Declarative**: What, not how

### 3. **Why Traits?**
- **LLMAdapter Trait**: Easy to add new LLM providers
- **Tool Trait**: Uniform interface for all tools
- **Extensibility**: Plugins can implement traits

### 4. **Why Async?**
- **I/O Bound**: LLM APIs, tools, databases
- **Concurrency**: Handle multiple requests
- **Streaming**: Real-time responses

### 5. **Why Workspace?**
- **Modularity**: Independent crates
- **Reusability**: nexus-llm can be used standalone
- **Testing**: Test each crate in isolation
- **Compilation**: Faster incremental builds
